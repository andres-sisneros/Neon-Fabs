# Neon Fabs Playtest Checklist

Use this checklist for quick manual runs after UI or core-loop changes. The goal is to confirm the first session is readable without relying on explanatory UI copy.

## Fresh Profile

- [ ] Start a new prototype session.
- [ ] Confirm the intro splash reads as a game, not a developer dashboard.
- [ ] Dismiss the intro and confirm it does not return after refresh.
- [ ] Choose Chrome Pier as home.
- [ ] Repeat once with Orchid Sprawl as home.
- [ ] Confirm Profile reads as status, not a suggested task list.
- [ ] Confirm Contracts are the only explicit early guidance.

## Starter Fab Loop

- [ ] Open Fabs.
- [ ] Confirm only local city fabs appear as local operations.
- [ ] Confirm a fresh profile starts with four sealed starter prints.
- [ ] Open Print Bay.
- [ ] Confirm results are readable and sorted by rarity.
- [ ] Confirm inventory receives only what fits.
- [ ] Confirm blocked collection does not recharge battery.
- [ ] Confirm Patterns unlock after the first collection.

## Inventory And Market

- [ ] Open Inventory in the current city.
- [ ] Filter by category, rarity, and search.
- [ ] Sell one item to a local bid.
- [ ] Open the item's market book.
- [ ] Clear filters from empty results.

## Melds

- [ ] Open Melds in the home city.
- [ ] Confirm the first common pattern is ready after the guaranteed first collection.
- [ ] Fuse one ready pattern.
- [ ] Confirm battery capacity increases by one hour.
- [ ] Confirm Market, Map, and Roles unlock after the first pattern.

## Second Fab

- [ ] Open Fab Shop.
- [ ] Confirm current-city availability is clear.
- [ ] Rent or buy a second fab if credits allow.
- [ ] Confirm the fab appears in the install city only.
- [ ] Pick a print pattern and verify it persists.

## Dispatch

- [ ] Switch to Merchant.
- [ ] Send one vehicle with cargo to a connected city.
- [ ] Advance time globally from Admin.
- [ ] Confirm vehicle and cargo arrive in the destination inventory.
- [ ] Confirm role changing is blocked while profession route work is active.

## Phone Browser

- [ ] Load the app on a phone-width viewport.
- [ ] Confirm bottom navigation is usable.
- [ ] Confirm Profile, Fabs, Inventory, Market, and Contracts fit without horizontal scrolling.
- [ ] Confirm dense actions collapse into buttons or action sheets instead of long button rows.
